# 📈 주식 매매 관리 시스템(STMS) V2.0 사용자 설명서

## 1\. 프로그램 개요 및 필요성

### 1.1. 프로그램 소개

이 프로그램은 주식 매매 관리 시스템(Stock Trading Management System)으로, 사용자가 웹 기반의 인터페이스를 통해 주식 매매 내역을 관리하고, 현재가를 확인하며, 거래 의견을 얻을 수 있도록 설계된 웹 애플리케이션입니다. 개인 투자자가 복잡한 증권사 HTS 없이 로컬 환경에서 자신의 주식 포트폴리오를 효율적으로 추적하고 관리할 수 있도록 돕는 간단한 웹 기반 도구입니다.

### 1.2. 프로그램의 필요성 🤔

주식 투자에서 가장 어려운 것 중 하나는 **감정에 휘둘리지 않고 정해진 원칙을 지키는 것**입니다. 객관적인 기준 없이 감정에 따라 매수/매도 결정을 내릴 경우 비합리적인 투자가 될 수 있습니다. 또한, 여러 종목의 현재가를 일일이 확인하고 손익을 계산하는 것은 번거로운 작업입니다.

이 시스템은 이러한 과정을 자동화하고, 사용자가 미리 정해둔 기준에 따라 다음과 같은 가치를 제공함으로써 보다 체계적이고 합리적인 주식 관리를 가능하게 합니다.

  * **감정적 매매 방지**: 명확한 규칙에 따라 '매수', '매도' 의견을 시스템이 계산해주므로, 감정적인 판단을 배제하고 기계적인 대응을 할 수 있습니다.
  * **체계적인 관리**: 여러 종목의 매입가, 현재가, 고점, 손익률 등을 한눈에 파악하고 관리할 수 있습니다.
  * **매매 기록**: 모든 매수/매도 결정을 '주식 매매 일지'에 기록하여, 과거의 판단을 복기하고 투자 전략을 개선하는 데 도움을 줍니다.
  * **데이터 연동**: `yfinance` 라이브러리를 통해 보유 종목의 현재가와 실시간 환율 정보를 자동으로 업데이트합니다.

## 2\. 프로그램 구성 요소

이 시스템은 크게 프론트엔드와 백엔드, 두 가지 파일로 구성됩니다.

  * **`STMS.py` (백엔드 Back-end)**
이 파일은 http.server 모듈을 사용해 간단한 웹 서버를 구축하고 있습니다. 주요 기능은 다음과 같습니다.

    * 웹 서버 실행: run_server() 함수가 8000번 포트에서 서버를 시작하고, 자동으로 웹 브라우저를 열어 사용자에게 인터페이스를 제공합니다.

    * API 엔드포인트:
      * `/api/save`: 사용자의 주식 데이터를 JSON 파일로 저장합니다. 파일명과 저장 경로를 지정할 수 있는 기능을 제공합니다.
      * `/api/load & /api/load_file`: 저장된 JSON 파일을 불러와서 사용자에게 보여줍니다. 기본 폴더나 지정된 경로에서 파일을 로드할 수 있습니다.
      * `/api/files`: stock_data 폴더에 있는 파일 목록을 클라이언트에게 전달해주는 역할을 합니다.
      * `/api/update_prices`: yfinance 라이브러리를 사용하여 종목 코드를 기준으로 현재 주가를 가져와서 업데이트합니다. 미국 주식의 경우 환율을 적용하여 원화로 변환하는 로직도 포함되어 있습니다.
      * `/api/update_exchange_rate`: 마찬가지로 yfinance 라이브러리를 사용해 실시간 USD-KRW 환율을 조회합니다.
      * `/api/exit`: 프로그램을 안전하게 종료하는 기능입니다.

    * 데이터 처리: JSON 데이터를 파싱하고, 파일 입출력을 처리하며, 외부 라이브러리 (yfinance)를 호출하는 등 실질적인 데이터와 로직을 담당합니다.

  * **`index.html` (프론트엔드 Front-end)**
사용자에게 보여지는 모든 시각적 요소가 정의된 파일입니다.

    * UI 구성:
      * 헤더: 주식 추가, 업데이트, 삭제, 저장, 불러오기, 설정, 종료 버튼 등 주요 기능을 위한 제어판 역할을 합니다.
      * 섹션: '저장', '불러오기', '설정'과 같이 특정 기능을 위한 입력 폼들이 숨겨져 있다가 필요할 때 나타나는 구조로 설계되어 있습니다.
      * 탭 인터페이스: '보유 주식 목록'과 '주식 매매 일지'를 구분하여 보여주는 탭 방식의 UI를 사용해 여러 정보를 효율적으로 보여줍니다.
      * 테이블: 보유 주식 목록과 매매 일지 데이터를 표 형태로 깔끔하게 시각화합니다.
      * 모달: '주식 관리'와 '프로그램 종료'와 같이 사용자에게 추가적인 상호작용이나 확인이 필요할 때 오버레이 형태로 나타나는 팝업 창입니다.

  * **`script.js` (프론트엔드 로직 Front-end Logic)**
사용자와의 모든 상호작용을 처리하는 핵심 스크립트 파일입니다.

    * 상태 관리: stocks 배열과 tradingLog 배열을 전역 변수로 관리하여, 사용자가 입력한 주식 데이터와 매매 기록을 임시로 저장하고 화면에 렌더링합니다.
    * DOM 제어: document.getElementById와 같은 메서드를 사용해 HTML 요소를 찾아 내용을 변경하거나, 클래스를 추가/제거하는 등 UI를 동적으로 조작합니다.
    * 이벤트 리스너: 버튼 클릭(onclick), 키보드 입력(onkeypress), 폼 제출(onblur) 등 사용자의 모든 이벤트를 감지하고 그에 맞는 함수를 실행합니다.
    * 백엔드 통신: fetch() API를 이용해 STMS.py 서버의 API 엔드포인트에 요청을 보냅니다. 예를 들어, '현재가 업데이트' 버튼을 누르면 /api/update_prices 엔드포인트에 요청을 보내고, 서버로부터 받은 응답 데이터를 처리하여 화면을 갱신합니다.
    * 데이터 처리: 매입가, 현재가, 고점 등을 기반으로 손익, 매도/매수 의견 등을 계산하는 로직이 포함되어 있습니다.
    * 로컬 저장소: localStorage를 사용하여 브라우저에 주식 데이터와 설정값을 영구적으로 저장하여 페이지를 새로고침하거나 다시 방문해도 데이터가 유지되도록 합니다.

## 3\. 핵심 알고리즘: 원칙 기반 분할 매수/매도 전략

이 프로그램의 핵심은 사용자가 설정한 비율에 따라 '매도 시점'과 '추가 매수 시점'을 자동으로 계산하고 투자 의견을 제시하는 것입니다.

### 3.1. 기준점: 고점 (High Point)

  * 주식을 매입한 시점의 '매입가'가 초기 '고점'이 됩니다.
  * 이후 '현재가'가 이전 '고점'을 돌파하면, 새로운 '현재가'가 새로운 '고점'으로 갱신됩니다.
  * 고점은 하향 조정되지 않고, 기록된 최고 가격을 계속 유지합니다.

### 3.2. 매도가 / 추가 매수가 계산

사용자가 '설정'에서 지정한 비율(기본값: 매도 -8%, 추가 매수 -5%)을 사용하여 계산합니다.

  * **매도가 (Sell Price)**: 이익을 실현하거나 손실을 제한하기 위한 매도 기준가입니다.

$$
$$$$매도가 = 고점 \\times (1 - \\frac{매도\\ 비율}{100})
$$
$$$$
$$

  * **추가 매수가 (Buy More Price)**: 주가가 고점에서 약간 조정을 보였을 때 비중을 늘리기 위한 추가 매수 기준가입니다.

$$
$$$$추가\\ 매수가 = 고점 \\times (1 - \\frac{추가\\ 매수\\ 비율}{100})
$$
$$$$
$$

### 3.3. 투자 의견 결정 로직

위에서 계산된 가격들을 현재가와 비교하여 '매도', '매수', '유지' 의견을 제시합니다.

| 조건 | 의견 | 의미 |
| :--- | :--- | :--- |
| `현재가 < 매도가` | **매도** | 손절 또는 익절을 고려해야 하는 구간입니다. |
| `매도가 <= 현재가 < 추가매수가` | **매수** | 추가 매수를 고려해볼 수 있는 구간입니다. |
| `현재가 >= 추가매수가` | **유지** | 현재 포지션을 유지하며 추이를 지켜보는 관망 구간입니다. |

## 4\. 프로그램 작동 원리

1.  **서버 시작**: 사용자가 `STMS.py`를 실행하면 로컬 웹 서버가 시작되고 `index.html` 파일을 브라우저에 제공합니다.
2.  **클라이언트-서버 통신**: 사용자가 '현재가 업데이트' 버튼을 클릭하면, `index.html`의 JavaScript는 `fetch` API를 사용해 백엔드 서버(`/api/update_prices`)로 HTTP 요청을 보냅니다.
3.  **데이터 처리**: `STMS.py` 서버는 요청을 받아 `yfinance`로 가격 정보를 조회한 후, 처리 결과를 JSON 형식으로 프론트엔드에 응답합니다.
4.  **화면 업데이트**: `index.html`은 서버로부터 받은 응답을 바탕으로 주식 테이블을 다시 렌더링하여 사용자에게 최신 정보를 보여줍니다.
5.  **데이터 보존**: `index.html`은 주식 목록과 설정값을 브라우저의 \*\*로컬 스토리지(Local Storage)\*\*에 저장하여, 브라우저를 다시 열어도 데이터가 유지되도록 합니다. 또한, 파일 저장/불러오기 기능으로 데이터를 영구적으로 관리할 수 있습니다.

## 5\. 🛠️ 설치 및 실행 방법

### 5.1. 설치 준비

  * **Python 설치**: PC에 Python 3 버전이 설치되어 있어야 합니다.
  * **필요 라이브러리 설치**: 터미널(명령 프롬프트)을 열고 아래 명령어를 실행하여 `yfinance` 라이브러리를 설치합니다.
    ```bash
    pip install yfinance
    ```
  * **프로그램 파일 준비**: `index.html`과 `STMS.py` 파일을 동일한 폴더에 위치시킵니다.

### 5.2. 웹 서버 실행

  * 터미널에서 파일이 있는 폴더로 이동한 후, 아래 명령어를 입력하여 서버를 실행하거나, run_STMS.bat 파일을 클릭하여 실행합니다.
    ```bash
    python STMS.py
    ```
  * 서버가 정상적으로 실행되면 자동으로 기본 웹 브라우저에서 프로그램(`http://localhost:8000`)이 열립니다.

### 5.3. 폴더 구조

실행 후 다음과 같은 폴더 및 파일 구조가 생성될 수 있습니다:

```
프로그램 폴더/
├── README.md          # 사용자 설명서
├── run_STMS.bat       # 메인 서버 프로그램을 실행하기 위한 배치 프로그램
├── STMS.py            # 메인 서버 프로그램
├── index.html         # 웹 인터페이스
└── stock_data/        # 기본 데이터 저장 폴더
    └── stock_data_YYYYMMDD_HHMMSS.json
```

  * **보유 주식 목록 (데이터 테이블)**: 등록된 주식의 상세 정보를 표시하는 메인 화면입니다.
  * **주식 매매 일지**: 매수/매도 기록을 시간 순으로 보여주는 탭입니다.

#### 테이블 컬럼 이해

| 컬럼 | 설명 |
| :--- | :--- |
| **주식 이름** | 등록한 주식의 이름입니다. |

### 6.3. 실전 사용 예시 (시나리오)

#### 시나리오 1: 수익 상황에서 '매수' 의견 발생

> 삼성전자 매입가: 70,000원, 현재 고점: 80,000원, 현재가: 75,000원

  * **매도 기준가**: 73,600원 (80,000원 × (1 - 0.08))
  * **추가매수 기준가**: 76,000원 (80,000원 × (1 - 0.05))
  * **판단**:
      * 현재가(75,000원)는 매도가(73,600원)보다 높고, 추가매수가(76,000원)보다 낮습니다.
      * **의견**: "매수"가 표시됩니다.
  * **해석**: 현재 수익 상태이지만 고점 대비 일부 하락하여 추가 매수를 고려해 볼 수 있는 구간입니다. 신중한 접근이 필요합니다.

#### 시나리오 2: 손절 상황

> 카카오 매입가: 60,000원, 현재 고점: 65,000원, 현재가: 55,000원

  * **매도 기준가**: 59,800원 (65,000원 × (1 - 0.08))
  * **추가매수 기준가**: 61,750원 (65,000원 × (1 - 0.05))
  * **판단**:
      * 현재가(55,000원)가 매도 기준가(59,800원)보다 낮습니다.
      * **의견**: "매도"가 표시됩니다.
  * **해석**: 손실이 확대되고 있으므로 추가 하락을 방지하기 위해 손절(매도)을 즉시 검토해야 합니다.

### 6.4. 고급 활용 팁

  * **포트폴리오 관리**: 여러 종목을 등록하여 포트폴리오 전체를 동시에 모니터링하세요.
  * **백업 전략**: 매일 거래 종료 후 데이터를 저장하여 투자 히스토리를 보관하는 것을 권장합니다. 파일명의 타임스탬프로 시점별 데이터 관리가 용이합니다.
  * **성과 분석**: 저장된 과거 데이터와 현재 데이터를 비교하여 자신의 투자 패턴과 성공/실패 요인을 파악할 수 있습니다.
  * **리스크 관리**: 손절 신호 발생 시 즉시 검토하고, 시장 상황과 프로그램의 신호를 종합하여 최종 판단을 내리세요.
