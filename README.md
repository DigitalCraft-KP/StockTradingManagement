# StockTradingManagement
이 프로그램은 주식 매매 관리 시스템(Stock Trading Management System)으로, 사용자가 웹 기반의 인터페이스를 통해 주식 매매 내역을 관리하고, 현재가를 확인하며, 거래 의견을 얻을 수 있도록 설계된 웹 애플리케이션입니다. 
주식 매매 관리 시스템은 개인 투자자가 자신의 주식 포트폴리오를 효율적으로 관리할 수 있도록 돕는 간단한 웹 기반 도구입니다. 복잡한 증권사 애플리케이션이나 금융 소프트웨어 없이, 로컬 환경에서 직접 주식의 매입가, 현재가, 손익을 추적하고, 사전 설정된 규칙에 따라 매수 및 매도 의견을 시각적으로 제공합니다. 의견 정보를 활용하여 체계적인 투자 전략을 수립하고, 매수/매도 시점을 객관적으로 판단할 수 있도록 지원합니다.
복수의 주식 종목을 관리할 때, 각 종목의 현재가를 일일이 확인하고 손익을 계산하는 것은 번거로운 작업입니다. 또한, 객관적인 기준 없이 감정에 따라 매수/매도 결정을 내릴 경우 비합리적인 투자가 될 수 있습니다. 이 시스템은 이러한 과정을 자동화하고, 사용자가 미리 정해둔 기준에 따라 매매 의견을 제공함으로써 보다 체계적이고 합리적인 주식 관리를 가능하게 합니다.

## 1. 프로그램 구성 요소
이 시스템은 크게 두 가지 파일로 구성됩니다.
 - index.html: 사용자가 직접 상호작용하는 **프론트엔드(Front-end)** 를 담당합니다. HTML, CSS, JavaScript로 작성되었으며, 주식 목록을 표시하고, 사용자의 입력을 받아 백엔드와 통신하는 역할을 수행합니다.
   - 데이터 처리: yfinance 라이브러리를 사용해 주식 현재가, 환율 등의 금융 데이터를 실시간으로 조회합니다.
   - 파일 입출력: 사용자의 요청에 따라 .json 파일에 데이터를 저장하거나, 저장된 파일을 읽어와 프론트엔드에 전달합니다.

 - STMS.py: 서버 역할을 하는 **백엔드(Back-end)** 프로그램입니다. Python으로 작성되었으며, yfinance 라이브러리를 사용하여 주식 정보를 가져오고, 파일 입출력 및 데이터 처리를 담당합니다.
   - API Endpoint: 프론트엔드로부터 오는 요청을 처리하기 위한 여러 API 엔드포인트(예: /api/update_prices, /api/save)를 제공합니다.
   - 데이터 처리: yfinance 라이브러리를 사용해 주식 현재가, 환율 등의 금융 데이터를 실시간으로 조회합니다.
   - 파일 입출력: 사용자의 요청에 따라 .json 파일에 데이터를 저장하거나, 저장된 파일을 읽어와 프론트엔드에 전달합니다.


## 2. 주요 기능 및 흐름
### 2.1. 사용자 인터페이스 (index.html)
사용자는 브라우저에서 index.html 파일을 통해 시스템을 사용합니다.  이 인터페이스는 다음과 같은 기능을 제공합니다.

 - 주식 추가 및 관리: 주식 이름, 종목 코드, 매입가 등을 입력하여 새로운 주식을 목록에 추가하고, 기존 항목을 수정하거나 삭제할 수 있습니다.
 - 현재가 업데이트: '현재가 업데이트' 버튼을 클릭하면 등록된 모든 주식의 현재 가격을 자동으로 가져와 업데이트합니다.
 - 매매 설정: '설정' 섹션에서 매도 비율, 추가 매수 비율, 그리고 **환율(USD→KRW)**을 사용자가 직접 설정할 수 있습니다.
 - 데이터 저장 및 불러오기:
    - 저장: 현재 관리 중인 주식 데이터를 .json 파일로 저장합니다. 사용자는 파일명과 저장 경로를 지정할 수 있습니다.
    - 불러오기: 이전에 저장된 .json 파일을 불러와서 작업 내용을 복원합니다. 기본 저장 폴더의 목록에서 선택하거나, 직접 파일 경로를 입력하여 불러올 수 있습니다.

### 2.2. 백엔드 서버 (STMS.py)
STMS.py 파일은 로컬 컴퓨터에서 웹 서버를 실행하며, index.html에서 보낸 요청을 처리합니다.

 - 웹 서버 실행: http.server 모듈을 사용하여 로컬 서버(기본 포트 8000)를 엽니다. 서버가 실행되면 사용자의 기본 웹 브라우저가 자동으로 열립니다.
 - 데이터 처리 API:
    - GET /api/files: 기본 저장 폴더(stock_data)에 있는 .json 파일 목록을 사용자에게 반환합니다.
    - POST /api/save: 사용자가 입력한 데이터를 JSON 형식으로 변환하여 지정된 경로에 .json 파일로 저장합니다. 경로가 지정되지 않으면 stock_data 폴더에 타임스탬프 기반의 파일명으로 저장합니다.
    - POST /api/load: 기본 폴더의 특정 .json 파일을 읽어와서 사용자에게 반환합니다.
    - POST /api/load_file: 사용자가 지정한 임의의 경로에 있는 .json 파일을 읽어와 반환합니다.
    - POST /api/update_prices: yfinance 라이브러리를 사용해 주식 종목 코드에 해당하는 현재가를 가져옵니다. 미국 주식의 경우 설정된 환율을 적용하여 원화로 변환합니다.
    - POST /api/update_exchange_rate: yfinance를 통해 USD/KRW 환율을 실시간으로 가져와 반환합니다.
## 3. 핵심 알고리즘
### 3.1	고점(High Point) 계산
* 매입가를 초기 고점으로 설정
* 현재가가 매입가보다 높고, 기존 고점보다 높을 때만 고점 업데이트
* 고점은 하향 조정되지 않음 (최고점 유지)
  
### 3.2	매도가/추가매수가 계산
  •	매도가: 고점 × 0.92 (고점 대비 -8%)
  •	추가매수가: 고점 × 0.95 (고점 대비 -5%)
### 3.3	투자 의견 결정 로직
```
if 현재가 >= 추가매수가 AND 현재가 > 매도가:
    의견 = "매수"
elif 현재가 < 매도가:
    의견 = "매도"
else:
    의견 = "유지"
```

### 3.4	데이터 흐름
 * 사용자가 주식 정보 입력 (주식명, 매입가)
 * 시스템이 초기 데이터 설정 (현재가 = 매입가, 고점 = 매입가)
 * 실시간 주가 API 호출로 현재가 업데이트
 * 고점 및 매매 기준점 자동 재계산
 * 투자 의견 및 행동 지침 자동 생성
 * 데이터를 로컬 스토리지 및 파일에 저장

## 4. 프로그램 작동 원리
 - 서버 시작: 사용자가 STMS.py 파일을 실행하면, Python 스크립트가 로컬 웹 서버를 시작하고 index.html 파일을 제공합니다.
 - 클라이언트-서버 통신: index.html에 있는 자바스크립트 코드는 사용자의 행동(예: '현재가 업데이트' 버튼 클릭)에 따라 fetch API를 사용해 백엔드 서버의 특정 API(예: /api/update_prices)로 HTTP 요청을 보냅니다.
 - 데이터 처리: STMS.py 서버는 요청을 받아 필요한 데이터 처리(예: yfinance를 사용한 가격 정보 조회)를 수행하고, 처리 결과를 JSON 형식으로 다시 프론트엔드로 보냅니다.
 - 화면 업데이트: index.html은 서버로부터 받은 응답을 바탕으로 주식 테이블을 다시 렌더링(rendering)하여 사용자에게 최신 정보를 보여줍니다.
 - 데이터 보존: index.html은 사용자가 추가한 주식 목록과 설정값을 **로컬 스토리지(Local Storage)**에 저장하여 브라우저를 닫았다가 다시 열어도 데이터가 유지되도록 합니다. 또한, 파일 저장/불러오기 기능을 통해 데이터를 영구적으로 관리할 수 있습니다.

## 5	🛠️ 프로그램 설치 및 실행 방법
### 5.1	설치 준비
* Python 설치: PC에 Python 3가 설치되어 있어야 합니다.
* 필요 라이브러리 설치 : Yahoo 서버를 통해 현재가 정보를 읽어오기 위해 아래 라이브러리가 필요합니다.
```
pip install yfinance
```
* 프로그램 파일 준비: index.html과 STMS.py 파일을 동일한 폴더에 둡니다.

### 5.2	웹 서버 실행
* 터미널(명령 프롬프트)을 열고 파일이 있는 디렉터리로 이동한 후 다음 명령어를 입력합니다. 또는 run_STMS.bat 파일을 클릭합니다.
```
Python STMS.py
```
* 웹 브라우저 접속: 서버가 실행되면 터미널에 표시된 주소(예: http://localhost:8000)를 웹 브라우저에 입력하여 접속합니다.

### 5.3	폴더 구조
실행 후 다음과 같은 폴더 구조가 생성됩니다:
```
프로그램 폴더/
├── run_STMS.bat       # Command 창에서 실행하기 위한 배치 파일
├── STMS.py            # 메인 서버 프로그램
├── index.html         # 웹 인터페이스
└── stock_data/        # 기본 데이터 폴더
    ├── stock_data_YYYYDDMM_HHMMSS.json
```

## 6	📝 프로그램 사용 방법
### 6.1	기본 사용 흐름
#### 6.1.1	초기 설정
* 프로그램 실행
* 브라우저에서 시스템 접속
 
#### 6.1.2	주식 등록
* "+ 주식 추가" 버튼 클릭
* 주식 이름 / 매입가를 입력하고, “추가” 버튼 클릭 (예: "삼성전자 / 70000")한 후, 테이블에 새 주식 정보 표시 확인

#### 6.1.3	현재가 업데이트 및 행동
* 업데이트 주기는 일정한 기간을 두어 변동이 심한 경우에 급하게 대응하지 않도록 합니다.
* 하루 5%이상 급등락을 하는 경우, 업데이트를 통해 행동 의견 정보를 받아서 주식 거래 진행합니다.
* 자동 업데이트 사용 방법

#### 6.1.4	주식 종목 관리 추천
* 추가 매수를 하는 경우 매입가를 변경하여 바로 적용할 수 있습니다.
* 매도를 하는 경우, 매도 종목을 삭제하여 미련을 버리는 것이 좋습니다.

### 6.2	주요 기능 사용법
#### 6.2.1	주식 추가
화면 상단의 "+ 주식 추가" 버튼을 클릭하여 주식 이름, 주식 종목 코드(Ticker), 매입가를 입력하고 "추가" 버튼을 누릅니다.

#### 6.2.2	데이터 테이블 확인
추가된 주식은 테이블에 표시됩니다. 각 항목은 다음과 같은 정보를 나타냅니다.
* 매입가: 사용자가 입력한 주식의 매입 단가.
* 현재가: 자동으로 업데이트되거나 수동으로 입력한 현재 주가.
* 고점: 매입 후 기록한 최고 가격.
* 매도: 고점의 92% 가격 (고점 -8%). -> 기본값은 -8%이나, 사용자가 변경할 수 있음.
* 추가매수: 고점의 95% 가격 (고점 -5%). -> 기본값은 -5%이나, 사용자가 변경할 수 있음.
* 손익: 현재가 대비 매입가의 이익 또는 손실 여부.
* 의견: 현재 주가 위치에 따른 '매수', '매도', '유지' 의견.
* 행동: '익절', '손절', '기다림'과 같은 최종 행동 지침.

##### 6.2.3	가격 업데이트
* 수동 입력: ✏️ 아이콘을 클릭하여 새로운 가격을 입력할 수 있습니다. 이 가격은 매입 후, 추가 매수 및 매도의 기준점이 되는 고점의 가격을 업데이트하는 데에 사용됩니다.

#### 6.2.4	데이터 저장/불러오기
* 저장: "💾 저장" 버튼을 클릭하여 현재 테이블의 데이터를 JSON 파일로 저장합니다.
* 불러오기: "📁 불러오기" 버튼을 클릭하여 이전에 저장한 파일 목록에서 원하는 파일을 선택해 데이터를 복원합니다.
 
 
 
#### 6.2.5	주식 삭제
* 테이블의 "삭제" 버튼을 클릭하여 해당 주식 항목을 삭제할 수 있습니다.
 
### 6.3	정보 해석 방법
#### 6.3.1	테이블 컬럼 이해
* 주식 이름: 등록한 주식명
* 매입가: 최초 구매한 가격
* 현재가: 현재 시장 가격 (실시간/수동 업데이트)
* 고점: 매입 후 기록된 최고가
* 매도: 고점의 -8% 가격 (손절/익절 기준)
* 추가매수: 고점의 -5% 가격 (추가 매수 기준)
* 손익: 매입가 대비 현재 수익/손실 상태
* 의견: 현재 상황에서의 매매 권장사항
* 행동: 최종 투자 행동 지침

#### 6.3.2	색상 코딩 시스템
* 🟢 녹색: 이익, 매수 관련 정보
* 🔴 빨간색: 손해, 매도, 손절 관련 정보
* 🟠 주황색: 익절 관련 정보
* 🔵 파란색: 추가매수 관련 정보
* 🟣 보라색: 고점 관련 정보
* ⚫ 회색: 중립, 기다림 관련 정보

#### 6.3.3	투자 의사결정 가이드

매수 신호:
* 의견: "매수", 행동: "기다림"
* 현재가가 추가매수 구간에 위치
* 추가 매수 검토 시점

매도 신호:
* 의견: "매도", 행동: "익절" 또는 "손절"
* 현재가가 고점 대비 8% 하락
* 즉시 매도 검토 시점

보유 신호:
* 의견: "유지", 행동: "기다림"
* 현재 상황에서는 매수/매도 모두 부적절
* 상황 변화 관찰 시점

### 6.4	실전 사용 예시
#### 6.4.1	시나리오 1: 수익 상황
```
삼성전자 매입가: 70,000원
현재가: 75,000원 (수동 입력)
고점: 80,000원
매도: 73,600원 (80,000 × 0.92)
추가매수: 76,000원 (80,000 × 0.95)
```
→ 손익: 이익 (+5,000원, +7.14%)
→ 의견: 매수 (현재가 75,000 > 매도가 73,600)
→ 행동: 기다림
* 해석: 현재 수익 상태이며, 추가 상승 여력이 있어 보유 권장
 
#### 6.4.2	시나리오 2: 손절 상황
```
카카오 매입가: 60,000원
현재가: 55,000원 (실시간 업데이트)
고점: 65,000원
매도: 59,800원 (65,000 × 0.92)
추가매수: 61,750원 (65,000 × 0.95)
```
→ 손익: 손해 (-5,000원, -8.33%)
→ 의견: 매도 (현재가 55,000 < 매도가 59,800)
→ 행동: 손절
* 해석: 손실이 확대되고 있어 추가 하락 방지를 위한 매도 검토 필요
